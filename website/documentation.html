Getting Started

Welcome to the SolarBot documentation! This guide will help you set up and configure your Solana arbitrage trading bot for optimal performance.

Prerequisites

Before you begin, make sure you have the following:

  • Node.js (v18 or higher) installed on your system
  • A Solana wallet with SOL for transaction fees
  • USDC or WSOL tokens for trading
  • A Telegram bot token (obtained from @BotFather)
  • Basic knowledge of command line interfaces

System Requirements

SolarBot can run on most modern computers and servers with the following minimum specifications:

  • 1GB RAM (2GB recommended)
  • 10GB storage space
  • Stable internet connection
  • Linux, macOS, or Windows operating system

Note

For optimal performance, we recommend running SolarBot on a VPS (Virtual Private Server) to ensure 24/7 operation. See the VPS Deployment section for more details.

Installation

Follow these steps to install SolarBot on your system:

Step 1: Clone the Repository

git clone https://github.com/yourusername/solana-telegram-arb-bot.git
cd solana-telegram-arb-bot

Step 2: Install Dependencies

npm install

Step 3: Create Environment File

Create a .env file based on the provided example:

cp .env.example .env

Step 4: Configure Environment Variables

Edit the .env file with your credentials:

# Telegram Configuration
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_telegram_chat_id

# Solana Wallet Configuration
PRIVATE_KEY=[your_wallet_private_key_array]

# Google Sheets Configuration (Optional)
SHEET_ID=your_google_sheet_id

# Bot Operation Mode
SIMULATION=true

# RPC Endpoint (Optional)
RPC_ENDPOINT=https://api.mainnet-beta.solana.com

Warning

Never share your private key or .env file with anyone. Keep your credentials secure at all times.

Step 5: Start the Bot

npm start

If everything is configured correctly, you should see output indicating that the bot has started successfully.

Configuration

SolarBot offers extensive configuration options to customize its behavior according to your trading preferences.

Configuration Files

The main configuration files are located in the config directory:

  • settings.js - General bot settings
  • tokens.js - Token addresses and pairs to monitor

Settings Configuration

The settings.js file contains various settings that control the bot's behavior:

Scanning Settings

scanning: {
  // Interval between scans in milliseconds
  interval: 30000,

  // Maximum number of opportunities to process in a single scan
  maxOpportunities: 5,

  // Whether to automatically start scanning on bot startup
  autoStart: true,

  // Maximum number of concurrent requests to avoid rate limiting
  maxConcurrentRequests: 3
}

Trading Settings

trading: {
  // Default minimum profit percentage to consider an opportunity valid
  defaultMinProfitPercent: 0.5,

  // Maximum slippage percentage allowed for trades
  maxSlippagePercent: 1.0,

  // Default amount to use for price quotes (in USDC)
  defaultQuoteAmount: 100,

  // Maximum amount to trade in a single transaction (in USDC)
  maxTradeAmount: 1000,

  // Whether to automatically execute trades when opportunities are found
  autoExecuteTrades: false,

  // Minimum time between trades in milliseconds (to avoid spamming)
  minTimeBetweenTrades: 60000
}

Risk Management Settings

riskManagement: {
  // Maximum daily trading volume (in USDC)
  maxDailyVolume: 10000,

  // Maximum number of trades per day
  maxDailyTrades: 50,

  // Maximum percentage of wallet to use for trading
  maxWalletPercentage: 50,

  // Minimum wallet balance to maintain (in SOL)
  minWalletBalance: 0.1,

  // Maximum loss percentage before pausing trading
  maxLossPercentage: 5.0,

  // Whether to enable circuit breaker (pause trading after consecutive losses)
  enableCircuitBreaker: true,

  // Number of consecutive losses to trigger circuit breaker
  circuitBreakerThreshold: 3
}

Telegram Setup

SolarBot uses Telegram for command control and notifications. Follow these steps to set up your Telegram bot:

Step 1: Create a Telegram Bot

  1. Open Telegram and search for @BotFather
  2. Start a chat with BotFather and send the command /newbot
  3. Follow the prompts to name your bot and create a username for it
  4. Once created, BotFather will provide you with a token (e.g., 123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ)
  5. Copy this token and add it to your .env file as TELEGRAM_BOT_TOKEN

Step 2: Get Your Chat ID

  1. Start a chat with your newly created bot
  2. Send any message to the bot
  3. Start the bot with npm start
  4. Send the command /chatid to your bot
  5. The bot will respond with your Chat ID
  6. Copy this ID and add it to your .env file as TELEGRAM_CHAT_ID

Step 3: Configure Bot Commands (Optional)

You can set up command suggestions for your bot by sending the following to BotFather:

  1. Send /mybots to BotFather
  2. Select your bot
  3. Click "Edit Bot" > "Edit Commands"
  4. Send the following list of commands:
start - Start the bot and show available commands
ping - Check if the bot is running
chatid - Get your chat ID
live - Enable live trading mode
simulate - Enable simulation mode
pause - Pause background scanning
resume - Resume background scanning
setprofit - Set minimum profit percentage
status - Get current bot status
summary - Get performance summary
opportunities - View recent opportunities
trades - View recent trades
settings - View current settings
help - Show help message

Note

For security reasons, only the chat ID specified in your .env file will be able to control the bot. This prevents unauthorized access to your trading bot.

VPS Deployment

For optimal performance, we recommend deploying SolarBot on a Virtual Private Server (VPS) to ensure 24/7 operation.

Recommended VPS Providers

  • DigitalOcean
  • AWS Lightsail
  • Linode
  • Vultr

Minimum VPS Specifications

  • 1 CPU core
  • 2GB RAM
  • 20GB SSD storage
  • Ubuntu 20.04 LTS or later

Deployment Steps

Step 1: Set Up Your VPS

  1. Create an account with your chosen VPS provider
  2. Create a new VPS instance with Ubuntu 20.04 LTS
  3. Set up SSH access to your VPS

Step 2: Install Dependencies

sudo apt update
sudo apt upgrade -y
sudo apt install -y curl git build-essential

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Verify installation
node -v
npm -v

Step 3: Clone and Set Up the Bot

git clone https://github.com/yourusername/solana-telegram-arb-bot.git
cd solana-telegram-arb-bot
npm install

# Create and edit .env file
cp .env.example .env
nano .env

Step 4: Install PM2 for Process Management

sudo npm install -g pm2

# Start the bot with PM2
pm2 start server.js --name solarbot

# Configure PM2 to start on boot
pm2 startup
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u $(whoami) --hp $(echo $HOME)
pm2 save

Step 5: Monitor Your Bot

# View logs
pm2 logs solarbot

# Monitor process
pm2 monit

# Restart the bot
pm2 restart solarbot

# Stop the bot
pm2 stop solarbot

Tip

Professional and Enterprise tier subscribers receive dedicated VPS setup assistance. Contact our support team for help with your deployment.

Frequently Asked Questions

How much profit can I expect from using SolarBot?

Profit varies based on market conditions, your settings, and the subscription tier you choose. Typically, users see opportunities ranging from 0.5% to 3% per trade. However, we cannot guarantee specific returns as cryptocurrency markets are volatile.

Is my wallet secure when using SolarBot?

Yes, your wallet security is our top priority. SolarBot runs on your own VPS or local machine, and your private keys never leave your environment. We do not have access to your funds or private keys at any time.

What happens if the bot encounters an error?

SolarBot has built-in error handling and will attempt to recover from most errors automatically. If a critical error occurs, the bot will send an alert to your Telegram chat and log the error for troubleshooting.

Can I run multiple instances of SolarBot?

Yes, Professional and Enterprise tier subscribers can run multiple instances of SolarBot with different configurations. This allows you to implement various strategies simultaneously.

How much SOL do I need to keep in my wallet?

We recommend maintaining at least 0.1 SOL in your wallet for transaction fees. The exact amount depends on your trading volume and the current network congestion.

What RPC endpoint should I use?

While SolarBot works with public RPC endpoints, we recommend using a dedicated RPC service like Helius or QuickNode for better performance and reliability, especially for Advanced and Professional tier users.

How do I update SolarBot to the latest version?

To update SolarBot, pull the latest changes from the repository and restart the bot:

git pull
npm install
pm2 restart solarbot

What should I do if I encounter a problem not covered in the documentation?

If you encounter an issue not covered in the documentation, please contact our support team through the Contact page or join our Telegram community for assistance.
